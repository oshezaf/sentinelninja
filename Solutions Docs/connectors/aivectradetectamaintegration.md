[Vectra AI Detect](https://www.vectra.ai/) is an AI-driven network detection and response (NDR) platform that identifies and prioritizes cyberattacks in real time across cloud, data center, and enterprise environments.

The connector uses the Azure Monitor Agent (AMA) to collect CEF logs, routing them via a Data Collection Rule (DCR) to the CommonSecurityLog table ([learn more about CEF via AMA](https://learn.microsoft.com/azure/sentinel/cef-syslog-ama-overview)).

To collect Vectra AI Detect logs into Microsoft Sentinel, navigate in the Vectra UI to Settings > Notifications > Edit syslog configuration. Add a new destination with port 514, UDP, CEF format; enable all log types; and test the connection.

## Tables Ingested

This connector ingests data into the following tables:

| Table | Selection Criteria | Transformations | Ingestion API | Lake-Only |
|:------|:-------------|:---------------:|:-------------:|:---------:|
| [`CommonSecurityLog`](../tables/commonsecuritylog.md) | `DeviceEventClassID == "hsc"`<br>`DeviceEventClassID !in "health,audit,campaigns,hsc,asc"`<br>`DeviceProduct == "X Series"`<br>`DeviceVendor == "Vectra Networks"` | ✓ | ✓ | ✓ |
