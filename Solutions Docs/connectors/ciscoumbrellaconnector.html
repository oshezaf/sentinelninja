The Cisco Cloud Security solution for Microsoft Sentinel enables you to ingest Cisco Secure Access and Cisco Umbrella logs stored in Cisco-managed Amazon S3 Bucket into Microsoft Sentinel using the Amazon S3 REST API. Refer to Cisco Cloud Security log management documentation for more information.

Setup Instructions

⚠️ Note: These instructions were automatically generated from the connector's user interface definition file using AI and may not be fully accurate. Please verify all configuration steps in the Microsoft Sentinel portal.

1. Configuration of the Cisco Cloud Security Logs Collection

NOTE: This connector has been updated to support Cisco Cloud Security log schema version 15. See documentation and follow the instructions to set up logging and obtain the necessary credentials and information. Once you have set up the Cisco-managed S3 Bucket, you will be given three pieces of information: Data Path, Access Key, and Secret Key. WARNING: For Cisco-managed S3 buckets, the Secret Key expires every 90 days. You are responsible for rotating the Secret Key in the Cisco dashboard and reconnecting this connector with the new key before expiration. For Access Key and Secret Key, you can paste them into the following inputs. For the other required fields, you will retrieve them from Data Path. For example, the Data Path provided by Cisco is cisco-managed-us-west-1/2003477-12345. For the AWS S3 Bucket Name, the input S3 Bucket Name will be cisco-managed-us-west-1. The S3 Bucket Region will be us-west-1. The S3 Bucket Prefix will be 2003477-12345. To enable stream-based collection, click Add new collector, choose a Data type, and provide the AWS details.
